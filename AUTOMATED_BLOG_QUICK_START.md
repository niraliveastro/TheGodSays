# Quick Start: Automated Blog Generation

## üöÄ Setup in 5 Minutes

### Step 1: Add Environment Variables

Add to `.env.local`:

```env
# Required
OPENAI_API_KEY=your-openai-api-key-here
CRON_SECRET=your-random-secret-key-here

# Optional (defaults shown)
BLOG_GEN_MAX_PER_RUN=10
BLOG_GEN_AI_MODEL=gpt-4o-mini
BLOG_GEN_DELAY_MS=2000
```

**Also add these to Vercel Environment Variables** (Settings ‚Üí Environment Variables)

### Step 2: Verify Configuration

The system is pre-configured to:
- Generate blogs daily at 2 AM UTC
- Create 10 blogs per run
- Use GPT-4o-mini (cost-effective high-quality model)
- Auto-publish all generated blogs

### Step 3: Test Locally (Optional)

```bash
# Test with dry run (no actual blogs created)
curl "http://localhost:3000/api/cron/generate-blogs?max=2&dryRun=true"

# Or generate 2 test blogs
curl "http://localhost:3000/api/cron/generate-blogs?max=2" \
  -H "Authorization: Bearer YOUR_ADMIN_PASSCODE"
```

### Step 4: Deploy to Vercel

```bash
git add .
git commit -m "Add automated blog generation system"
git push
```

Vercel will automatically:
- Deploy the cron job
- Start generating blogs daily

### Step 5: Verify It Works

1. **Check Vercel Cron**: Dashboard ‚Üí Your Project ‚Üí Cron Jobs
2. **Check Generated Blogs**: Visit `/admin/blog` after first run
3. **Check Sitemap**: Visit `/sitemap.xml` (blogs auto-included)

## üéõÔ∏è Adjust Settings

### Change Publishing Rate

Edit `vercel.json`:

```json
{
  "crons": [{
    "path": "/api/cron/generate-blogs?max=5",  // Reduce to 5/day
    "schedule": "0 2 * * *"                     // Daily at 2 AM
  }]
}
```

### Change AI Model

In `.env.local`:

```env
BLOG_GEN_AI_MODEL=gpt-4o        # Higher quality, more expensive
BLOG_GEN_AI_MODEL=gpt-3.5-turbo # Lower cost, good quality
```

### Generate More Per Run

```env
BLOG_GEN_MAX_PER_RUN=20  # Generate 20 blogs per cron run
```
